# Автоматизований hardening веб-сервера (Ubuntu 24.04 LTS)

Цей проєкт містить shell-скрипт `hardening.sh`, призначений для базового налаштування безпеки (hardening) щойно встановленого веб-сервера на базі ОС Ubuntu 24.04 LTS. Скрипт працює автоматично, вимагає мінімум втручання та не має зовнішніх залежностей, окрім стандартних пакетів Ubuntu.

## Основні можливості

- **Автоматичне встановлення залежностей**: Nginx/Apache, MariaDB/MySQL, vsftpd, UFW, Fail2Ban, rkhunter.
- **Безпека SSH**:
  - Зміна стандартного порту (за замовчуванням 2222).
  - Відключення входу для `root` та авторизації за паролем.
  - Генерація або копіювання SSH-ключів для `admin`-користувача.
  - Захисний механізм ("Dead Man's Switch") для відкоту налаштувань, якщо доступ буде втрачено.
- **Захист Веб-сервера (Nginx/Apache)**: додавання security headers, приховування версій (`server_tokens off`), відключення відображення вмісту директорій (directory listing).
- **Безпека Баз Даних (MySQL/MariaDB)**: блокування віддаленого доступу `root`, видалення тестової БД, обмеження з'єднань лише через `127.0.0.1`.
- **Безпека FTP (vsftpd)**: використання chroot, налаштування TLS/FTPS (з Let's Encrypt або self-signed сертифікатом), обмеження діапазону PASV портів.
- **Мережевий захист**: скидання та налаштування firewall (UFW) з політикою Default Deny, встановлення Fail2Ban.
- **Системні політики**: конфігурація `login.defs`, обмеження тайм-ауту сесій терміналу, встановлення антивірусного сканера (`rkhunter`).

## Використання

### Вимоги
- Операційна система: свіжа інсталяція **Ubuntu 24.04 LTS**.
- Права `root` (або виконання через `sudo`).

### Базовий запуск
```bash
sudo bash hardening.sh
```

### Доступні опції

| Опція | Опис |
|---|---|
| `--ssh-port PORT` | Задати нестандартний порт для SSH (від 1 до 65535, за замовчуванням: 2222). |
| `--admin-user USER` | Задати ім'я non-root користувача для адміністрування (буде створений, якщо відсутній). |
| `--dry-run` | Запустити скрипт у режимі симуляції без внесення реальних змін у систему. |
| `--yes` | Автоматичне підтвердження першого запуску (для неінтерактивного режиму). |
| `--no-ufw-reset` | Зберегти наявні правила UFW замість скидання їх на стандартні. |
| `--no-upgrade` | Пропустити етап виконання `apt-get upgrade`. |
| `--no-strict-ssh` | Вимкнути застосування додаткових жорстких політик SSH, що рекомендовані Lynis. |
| `--no-malware-scanner` | Не встановлювати та не ініціалізувати сканер rkhunter. |
| `-h, --help` | Вивести довідку командного рядка. |

### Приклади

Симуляція дій (безпечний перегляд):
```bash
sudo bash hardening.sh --dry-run
```

Запуск зі зміною порту SSH на 3344 та вказанням адміністратора:
```bash
sudo bash hardening.sh --ssh-port 3344 --admin-user myadmin
```

## Зберігання бекапів та логів

- Всі внесені зміни логуються. Файл логу створюється за шляхом: `/var/log/hardening_YYYYMMDD_HHMMSS.log`
- Скрипт автоматично робить резервні копії всіх конфігурацій, які він змінює. Вони зберігаються у директорії: `/var/backups/hardening_YYYYMMDD_HHMMSS/`
- Якщо скрипт виявить синтаксичну помилку після модифікації (наприклад, у конфігу SSH або Nginx), він автоматично відновить оригінальний конфіг з бекапу.
